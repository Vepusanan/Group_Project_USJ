# Startup Discovery Workflow (Investor Perspective)

This document describes the **implemented** investor startup discovery flow in the platform.

---

## 1. Access Discovery Feed

1. Investor navigates to the **Startups** discovery page (`/startups`).
2. The feed displays startup cards in a **grid** (default) or **list** view.
3. **Default sort is Best Match** (`sort=match_score`) for investors — compatibility score descending.
4. Results are **paginated** (9 startups per page).

Match scores use a **rule-based compatibility model** across five dimensions: industry, stage, geography, investment size, and revenue status.

> Match scores are shown **only to logged-in investors**.

---

## 2. Review Cards and Filter

### Startup card contents

Each card shows: logo, company name, tagline, industry, location, funding stage, revenue status (when set), match score %, verification badge, connection status, and optional description snippet.

### Filters

| Filter | Implementation |
|--------|----------------|
| **Search** | Free text across name, tagline, description |
| **Industry** | Dropdown |
| **Stage** | Funding stage (Pre-seed → Series D+) |
| **Geography** | Country text field |
| **Revenue status** | Pre-revenue / Revenue-generating / Profitable |
| **Verification level** | All / Identity Verified+ / Business Verified only |

### Sort options

Best Match (default for investors), Newest First, Alphabetical, Recently Updated.

Investors can select up to **4 startups** for **comparison** (`/compare`).

---

## 3. Save to Watchlist

Investor clicks **Watchlist** on a card. The startup is saved without a connection. Access saved items at **`/watchlist`** in the navigation menu.

---

## 4. View Startup Profile

Investor clicks **View Profile** (`/startups/:id`).

- **Public sections** — identity, business overview, public links.
- **Private sections** (locked until **accepted connection**):
  - Funding details — “Connect with this startup to view funding details”
  - Documents — “Connect to access documents and resources”
  - Private contact — “Connect to view private contact details”

---

## 5. Compatibility Match Explanation

On the startup profile page, investors see:

- **Match score percentage** badge
- **Compatibility explanation paragraph** — generated by **Google Gemini** when `GEMINI_API_KEY` is set in `server/.env`; otherwise falls back to rule-based text from dimension scores

---

## 6. Initiate Connection or Pass

### Connect

- **Discovery cards** and **profile page** open a modal with an **optional intro message** (up to 300 characters).
- Connection request is sent with status **pending**.

### Pass

- **Pass** is available on discovery cards and the profile page **without connecting**.
- Passed startups are **removed from the discovery feed** and appear in the deal pipeline **Archived** column (`/pipeline`) with intent **Passed**.

### After connection accepted

- Deal pipeline card created at `CONNECTED` stage.
- Intent levels (Watching / Interested / Passed) can be set on pipeline cards and in **My Connections**.
